## <font style="color:rgb(0, 0, 0);">1.HTTP</font>
<font style="color:rgb(0, 0, 0);">客户端与服务端之间通过交换一个个独立的消息（而非数据流）进行通信。由客户端发出的消息被称作</font>_<font style="color:rgb(0, 0, 0);">请求</font>_<font style="color:rgb(0, 0, 0);">（request），由服务端发出的应答消息被称作</font>_<font style="color:rgb(0, 0, 0);">响应</font>_<font style="color:rgb(0, 0, 0);">（response）。</font>

### **<font style="color:rgb(0, 0, 0);">基本性质</font>**
<font style="color:rgb(0, 0, 0);">在客户端与服务器能够传递请求、响应之前，这两者间必须建立 TCP 连接，这个过程需要多次往返交互。</font>

<font style="color:rgb(0, 0, 0);">HTTP本身是无状态的,借助 HTTP Cookie 就可使用有状态的会话</font>

### <font style="color:rgb(0, 0, 0);">HTTP控制</font>
缓存:http可以控制文档如何被缓存

开放同源限制:使用http的标头可以开放同源限制

认证:使用HTTP cookie设置一个特定的会话

_<font style="color:rgb(0, 0, 0);">会话</font>_<font style="color:rgb(0, 0, 0);">：使用 HTTP Cookie 可以利用服务端的状态将不同请求联系在一起。</font>

### <font style="color:rgb(0, 0, 0);">HTTP流</font>
<font style="color:rgb(0, 0, 0);">打开一个 TCP 连接</font>

<font style="color:rgb(0, 0, 0);">发送一个 HTTP 报文</font>

<font style="color:rgb(0, 0, 0);">读取服务端返回的报文信息</font>

<font style="color:rgb(0, 0, 0);">关闭连接或者为后续请求重用连接</font>

HTTP报文有请求与响应两种类型

+ <font style="color:rgb(0, 0, 0);">要获取的那个资源的路径——去除了当前上下文中显而易见的信息之后的 URL，比如说，它不包括</font>[<font style="color:rgb(4, 76, 159);">协议</font>](https://developer.mozilla.org/zh-CN/docs/Glossary/Protocol)<font style="color:rgb(0, 0, 0);">（</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(237, 238, 240);">http://</font>`<font style="color:rgb(0, 0, 0);">）、</font>[<font style="color:rgb(4, 76, 159);">域名</font>](https://developer.mozilla.org/zh-CN/docs/Glossary/Domain)<font style="color:rgb(0, 0, 0);">（这里是</font><font style="color:rgb(0, 0, 0);"> </font>`<font style="color:rgb(0, 0, 0);background-color:rgb(237, 238, 240);">developer.mozilla.org</font>`<font style="color:rgb(0, 0, 0);">），或是 TCP 的</font>[<font style="color:rgb(4, 76, 159);">端口</font>](https://developer.mozilla.org/zh-CN/docs/Glossary/Port)<font style="color:rgb(0, 0, 0);">（这里是</font><font style="color:rgb(0, 0, 0);"> </font>`<font style="color:rgb(0, 0, 0);background-color:rgb(237, 238, 240);">80</font>`<font style="color:rgb(0, 0, 0);">）。</font>
+ <font style="color:rgb(0, 0, 0);">HTTP 协议版本号。</font>
+ <font style="color:rgb(0, 0, 0);">为服务端表达其他信息的可选</font>[<font style="color:rgb(4, 76, 159);">标头</font>](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Reference/Headers)<font style="color:rgb(0, 0, 0);">。</font>

### 基于HTTP的API
[<font style="color:rgb(4, 76, 159);">Fetch API</font>](https://developer.mozilla.org/zh-CN/docs/Web/API/Fetch_API)<font style="color:rgb(0, 0, 0);"> 是基于 HTTP 的最常用 API，其可用于在 JavaScript 中发起 HTTP 请求。</font>

### <font style="color:rgb(0, 0, 0);">身份认证</font>
[<font style="color:rgb(4, 76, 159);">RFC 7235</font>](https://datatracker.ietf.org/doc/html/rfc7235)<font style="color:rgb(0, 0, 0);"> </font><font style="color:rgb(0, 0, 0);">定义了一个 HTTP 身份验证框架，服务器可以用来质询（</font>[<font style="color:rgb(4, 76, 159);">challenge</font>](https://developer.mozilla.org/zh-CN/docs/Glossary/Challenge)<font style="color:rgb(0, 0, 0);">）客户端的请求，客户端则可以提供身份验证凭据。</font>

<font style="color:rgb(0, 0, 0);">质询与响应的工作流程如下：</font>

1. <font style="color:rgb(0, 0, 0);">服务器端向客户端返回</font><font style="color:rgb(0, 0, 0);"> </font>[<font style="color:rgb(4, 76, 159);">401</font>](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Reference/Status/401)<font style="color:rgb(0, 0, 0);">（Unauthorized，未被授权的）响应状态码，并在</font><font style="color:rgb(0, 0, 0);"> </font>[<font style="color:rgb(4, 76, 159);">WWW-Authenticate</font>](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Reference/Headers/WWW-Authenticate)<font style="color:rgb(0, 0, 0);"> </font><font style="color:rgb(0, 0, 0);">响应标头提供如何进行验证的信息，其中至少包含有一种质询方式。</font>
2. <font style="color:rgb(0, 0, 0);">之后，想要使用服务器对自己身份进行验证的客户端，可以通过包含凭据的</font><font style="color:rgb(0, 0, 0);"> </font>[<font style="color:rgb(4, 76, 159);">Authorization</font>](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Reference/Headers/Authorization)<font style="color:rgb(0, 0, 0);"> </font><font style="color:rgb(0, 0, 0);">请求标头进行验证。</font>
3. <font style="color:rgb(0, 0, 0);">通常，客户端会向用户显示密码提示，然后发送包含正确的 </font>`<font style="color:rgb(0, 0, 0);background-color:rgb(237, 238, 240);">Authorization</font>`<font style="color:rgb(0, 0, 0);"> 标头的请求。\</font>

### HTTP Cookie
<font style="color:rgb(0, 0, 0);">HTTP Cookie（也叫 Web Cookie 或浏览器 Cookie）是服务器发送到用户浏览器并保存在本地的一小块数据。浏览器会存储 cookie 并在下次向同一服务器再发起请求时携带并发送到服务器上。通常，它用于告知服务端两个请求是否来自同一浏览器——如保持用户的登录状态。Cookie 使基于</font>[<font style="color:rgb(4, 76, 159);">无状态</font>](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Guides/Overview#http_%E6%98%AF%E6%97%A0%E7%8A%B6%E6%80%81%EF%BC%8C%E6%9C%89%E4%BC%9A%E8%AF%9D%E7%9A%84)<font style="color:rgb(0, 0, 0);">的 HTTP 协议记录稳定的状态信息成为了可能。</font>

<font style="color:rgb(0, 0, 0);">Cookie 主要用于以下三个方面：</font>

会话状态,个性化设置,浏览器行为跟踪,

<font style="color:rgb(0, 0, 0);">新的浏览器 API 已经允许开发者直接将数据存储到本地如使用 </font>[<font style="color:rgb(4, 76, 159);">Web storage API</font>](https://developer.mozilla.org/zh-CN/docs/Web/API/Web_Storage_API)<font style="color:rgb(0, 0, 0);">（</font>`<font style="color:rgb(0, 0, 0);background-color:rgb(237, 238, 240);">localStorage</font>`<font style="color:rgb(0, 0, 0);"> 和 </font>`<font style="color:rgb(0, 0, 0);background-color:rgb(237, 238, 240);">sessionStorage</font>`<font style="color:rgb(0, 0, 0);">）或 </font>[<font style="color:rgb(4, 76, 159);">IndexedDB</font>](https://developer.mozilla.org/zh-CN/docs/Web/API/IndexedDB_API)<font style="color:rgb(0, 0, 0);"> 。</font>

<font style="color:rgb(0, 0, 0);">服务器收到 HTTP 请求后，服务器可以在响应标头里面添加一个或多个 </font>[<font style="color:rgb(4, 76, 159);">Set-Cookie</font>](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Reference/Headers/Set-Cookie)<font style="color:rgb(0, 0, 0);"> 选项</font>

<font style="color:rgb(0, 0, 0);">浏览器收到响应后通常会保存下 Cookie，并将其放在 HTTP </font>[<font style="color:rgb(4, 76, 159);">Cookie</font>](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Reference/Headers/Cookie)<font style="color:rgb(0, 0, 0);"> 标头内，向同一服务器发出请求时一起发送。</font>

<font style="color:rgb(0, 0, 0);">一个简单的 Cookie</font>

```plain
Set-Cookie: <cookie-name>=<cookie-value>
```

<font style="color:rgb(0, 0, 0);">这指示服务器发送标头告知客户端存储一对 cookie：</font>

```plain
HTTP/1.0 200 OK
Content-type: text/html
Set-Cookie: yummy_cookie=choco
Set-Cookie: tasty_cookie=strawberry

[页面内容]
```

<font style="color:rgb(0, 0, 0);">现在，对该服务器发起的每一次新请求，浏览器都会将之前保存的 Cookie 信息通过</font><font style="color:rgb(0, 0, 0);"> </font>[<font style="color:rgb(4, 76, 159);">Cookie</font>](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Reference/Headers/Cookie)<font style="color:rgb(0, 0, 0);"> </font><font style="color:rgb(0, 0, 0);">请求头部再发送给服务器。</font>

<font style="color:rgb(0, 0, 0);">http</font><font style="color:rgb(0, 0, 0);">Copy</font>

```plain
GET /sample_page.html HTTP/1.1
Host: www.example.org
Cookie: yummy_cookie=choco; tasty_cookie=strawberry
```
